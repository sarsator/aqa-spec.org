AQA — AI Question Answer

Offene Spezifikation für strukturierte Q&A-Inhalte, optimiert für KI-Verarbeitung und Quellenangaben.

v1.2.2-draft GitHub MIT-Lizenz Schema.org-kompatibel

Das Problem

FAQ-Seiten sind allgegenwärtig, doch Schema.org FAQPage liefert KI-Systemen kaum Anhaltspunkte zur Beurteilung der Antwortqualität. Es fehlen fragenspezifische Datumsstempel, Änderungshistorien, quellenbasierte Belege je Antwort sowie Angaben zur Wartungshäufigkeit. Ein KI-Crawler kann nicht feststellen, ob eine Antwort gestern oder vor drei Jahren verfasst wurde – oder ob sie überhaupt noch gepflegt wird.

Die Lösung

AQA ist eine dokumentierte Konvention, die bestehende Schema.org-Typen (Article, FAQPage, Question, Answer) über einen benutzerdefinierten JSON-LD-Kontext um eine überschaubare Menge an Erweiterungseigenschaften ergänzt. Es werden keine neuen Typen eingeführt, bestehende Strukturen bleiben unverändert. Validatoren, die den AQA-Namensraum nicht kennen, ignorieren die Zusatzfelder stillschweigend. Die vollständige Abwärtskompatibilität bleibt erhalten.

Konformitätsstufen

Basic

Minimaler Aufwand

Jede Antwort hat ein bekanntes Alter und mindestens eine Quellenangabe. Das Dokument deklariert seine Sprache und seinen Autor.

Standard

Mäßiger Aufwand

Die Inhalte werden aktiv gepflegt, Änderungen werden in einem Changelog nachverfolgt, und der branchenspezifische Kontext ist durch Sektorklassifizierung und Beobachtungsquellen explizit angegeben.

Full

Erheblicher Aufwand

Vollständige Herkunftskette: wer jede Antwort verfasst hat, was sich geändert hat, wann, warum und aus welcher Quelle. Fragenspezifische Autorenschaft und versionierte Changelogs.

AQA Shield

Wenn jede Frage in einem Dokument sowohl aiUsagePolicy als auch contentSignature enthält, qualifiziert sich das Dokument für AQA Shield. Die Richtlinie erklärt die erlaubten KI-Nutzungen (RAG-Zitat, Training, Zusammenfassung, direktes Zitieren, kommerzielle Nutzung). Die Signatur ermöglicht eine SHA-256-Hash-Überprüfung gegen inhaltliche Verfälschung. Shield funktioniert auf jeder Konformitätsstufe als orthogonale Garantie, die rechtlichen Schutz und kryptografische Integrität vereint.

Schnellstart

Eine minimale AQA-Basic-Implementierung:

{
  "@context": [
    "https://schema.org",
    "https://aqa-spec.org/ns/context.jsonld"
  ],
  "@type": "Article",
  "headline": "Frequently Asked Questions",
  "author": {
    "@type": "Organization",
    "name": "Your Company"
  },
  "datePublished": "2024-01-15",
  "dateModified": "2026-03-20",
  "inLanguage": "en",
  "mainEntity": {
    "@type": "FAQPage",
    "mainEntity": [
      {
        "@type": "Question",
        "name": "Your question here?",
        "dateCreated": "2024-01-15",
        "dateModified": "2026-03-20",
        "citation": "https://source-url.com/document",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "Your answer here."
        }
      }
    ]
  }
}

Ihre Implementierung validieren:

pip install jsonschema requests
python validators/validate.py your-file.jsonld

Philosophie

AQA ist offen, dauerhaft kostenlos und herstellerunabhängig. Jeder kann es implementieren, validieren und erweitern. Die Spezifikation steht unter der MIT-Lizenz und wird gemeinschaftlich weiterentwickelt.

Das Ziel ist, für die KI-Sichtbarkeit das zu leisten, was OpenGraph für das Teilen in sozialen Netzwerken geleistet hat: Herausgebern einen einfachen, standardisierten Weg zu bieten, ihre Inhalte maschinenlesbar zu machen – ohne bestehende Strukturen zu beeinträchtigen.
